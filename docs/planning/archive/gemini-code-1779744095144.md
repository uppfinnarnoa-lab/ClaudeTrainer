### SYSTEM ARCHITECTURE & RESEARCH PROMPT: SPORTS SCIENCE ANALYTICS ENGINE

#### 1. OVERVIEW & CONTEXT
We are building a data science pipeline for a running analytics web application. The input is a massive dataset consisting of **2,800 historical running activities** containing time-series data from GPX files (latitude, longitude, elevation, timestamps) and synchronized Heart Rate (HR) data. 

The goal is to move away from generic, standardized percentage-based formulas (e.g., standard Riegel or max HR percentages) and instead extract highly personalized physiological thresholds (LT1, LT2), dynamic pace zones, and race predictions based on the user's actual field data and current training status.

---

#### 2. DATA PIPELINE & PREPROCESSING (GPX + HR)
The raw data is noisy due to GPS drops, traffic lights, and HR monitor lag/artifacts. Implement a robust data-cleaning pipeline.

* **Outlier & Spike Detection:**
    * Apply hard biological boundaries (e.g., $HR < 40$ or $HR > 220$).
    * Filter based on the rate of change (derivative) of heart rate: $\frac{\Delta HR}{\Delta t}$. If $\Delta HR > 10\text{ bpm/s}$, discard and interpolate.
* **Physiological Phase-Shift Compensation (HR Lag):**
    * HR response lags behind mechanical effort by roughly 30–45 seconds. 
    * To align effort with response, time-shift the HR time-series backwards (or movement data forward) by $t - 30$ to $t - 45$ seconds before performing correlation analysis.
* **Rolling Window Anomaly Filtering:**
    * Analyze data using a rolling 5-minute window. Discard segments that match any of the following noise criteria:
        1. High directional variance (sharp turns, indicating urban running/stoplights).
        2. High acceleration variance (short sprints where HR cannot reach steady-state).
        3. Extreme elevation volatility (technical trail running where foot placement drastically alters the HR-to-pace ratio).
    * *Goal:* Isolate a "Golden Dataset" of clean, steady-state running segments.

---

#### 3. FEATURE ENGINEERING
From the cleaned GPX data, calculate two primary metrics for every second:

1.  **Grade Adjusted Pace (GAP):** Calculate distance using the Haversine formula and compute instantaneous slopes ($\frac{\Delta \text{Elevation}}{\Delta \text{Distance}}$). Apply an established metabolic cost model (e.g., the Minetti formula) to normalize running pace on hilly terrain to flat-ground equivalent pace.
2.  **Running Power (Watts):** Implement a lightweight physics model (e.g., GOVSS or simplified Minetti power estimation) utilizing kinetic energy (acceleration/velocity) and potential energy (elevation changes) plus a constant for body mass and aerodynamic drag to derive a linear $Power \propto \text{Energy Expenditure}$ curve.

#### 4. CORE ALGORITHMIC CORE & THRESHold ESTIMATION

##### A. Physiological Anchor Points (HR)
Locate the specific heart rates for the Aerobic Threshold (LT1) and Anaerobic Threshold (LT2) using behavioral and structural mathematical markers:
* **LT1 Estimation via Aerobic Decoupling (Cardiovascular Drift):**
    * Scan for continuous, steady-state runs $>50\text{ minutes}$ with low GAP variance.
    * Split the steady portion into two halves (First Half: mins 5–25, Second Half: mins 25–45).
    * Calculate the ratio $\frac{HR}{GAP}$ for both halves. 
    * Find the highest HR where $\text{Drift} = \left(\frac{\text{Ratio B}}{\text{Ratio A}}\right) - 1 \le 0.03 \text{ to } 0.05$. When drift consistently exceeds 5%, the user has crossed LT1.
* **LT2 Estimation via Heart Rate Deflection Point (HRDP / Conconi Method):**
    * Map the steady-state $[HR, Power]$ or $[HR, GAP]$ pairs. 
    * Fit a linear regression line through the aerobic intensities (130–170 bpm). 
    * Locate the deflection point at high intensities where the actual HR consistently drops *below* the linear projection. This deflection point signifies LT2.

##### B. The Hybrid Pace-Zone Mapping (Current Fitness)
*Do not cluster raw historical pace directly, as pace changes with seasonality and fitness.*
* Filter the steady-state dataset to only include the **last 6–8 weeks** to represent current fitness.
* Perform a linear regression ($Y = mX + c$) where $X = HR$ and $Y = GAP$.
* Map the statistically derived $HR_{LT1}$ and $HR_{LT2}$ points through this current fitness regression model to output the dynamic, current pace targets:
    $$GAP_{LT1} = m \cdot (HR_{LT1}) + c$$
    $$GAP_{LT2} = m \cdot (HR_{LT2}) + c$$

##### C. Dynamic Training Zones Definition
Generate 5 training zones based on the mapped paces:
* **Zone 1 (Recovery):** Slower than $GAP_{LT1} + 60\text{ s/km}$
* **Zone 2 (Aerobic Base / LT1 Training):** From $GAP_{LT1} + 30\text{ s/km}$ to $GAP_{LT1}$
* **Zone 3 (Tempo / Marathon Pace):** From $GAP_{LT1}$ to $GAP_{LT2} + 15\text{ s/km}$
* **Zone 4 (Threshold / LT2 Training):** $GAP_{LT2} \pm 10\text{ s/km}$
* **Zone 5 (VO2max):** Faster than $GAP_{LT2} - 15\text{ s/km}$

#### 5. INDIVIDUALIZED RACE TIME PREDICTION (POWER-LAW)
Instead of using standard Riegel ($1.06$ exponent), profile the runner's unique fatigue decay rate.

1.  **Extract Recent Peak Performance (Power-Law):**
    * Scan the last 6–8 weeks of data. Extract the absolute best average GAP for set durations/distances (e.g., 1 km, 3 km, 5 km, 10 km).
2.  **Determine Fatigue Exponent ($k$):**
    * Plot these points in a log-log space: $X = \log(\text{Distance})$, $Y = \log(\text{Speed})$.
    * Fit a straight line. The slope of this line is the user's personalized fatigue exponent ($k$).
    * *Interpretation:* An endurance-optimized runner will yield a shallow slope ($k \approx -0.05$). An explosive, under-trained runner will yield a steep slope ($k \approx -0.12$).
3.  **Predict Target Race Pace:**
    * Given a recent baseline performance ($D_{\text{recent}}$, $V_{\text{recent}}$), calculate the predicted velocity ($V_{\text{target}}$) for a target distance ($D_{\text{target}}$) using the custom $k$:
        $$V_{\text{target}} = V_{\text{recent}} \cdot \left(\frac{D_{\text{target}}}{D_{\text{recent}}}\right)^k$$
